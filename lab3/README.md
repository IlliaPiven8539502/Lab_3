## Опис папок та файлів

### `src/main/java/com/messenger/models/`

Пакет моделей даних. Містить прості класи (POJO) без бізнес-логіки — лише поля, конструктори, гетери/сетери та метод `toString()` для JSON-серіалізації.

| Файл | Клас | Поля |
|---|---|---|
| `User.java` | `User` | `id`, `name` |
| `Message.java` | `Message` | `id`, `conversationId`, `senderId`, `text`, `createdAt`, `status` |
| `Conversation.java` | `Conversation` | `id`, `type` |

### `src/main/java/com/messenger/storage/`

Пакет сховища. Відповідає виключно за читання та запис даних у файли. Не містить бізнес-логіки.

| Файл | Відповідальність |
|---|---|
| `Database.java` | Зберігання у форматі JSON Lines (JSONL). Методи: `saveUser`, `loadUsers`, `findUserById`, `saveConversation`, `findConversationById`, `saveMessage`, `loadMessagesByConversation`, `updateMessageStatus` |

**Формат файлів (JSON Lines):** кожен рядок — окремий JSON-об'єкт:
```
{"id":"abc...","name":"Аліса"}
{"id":"def...","name":"Богдан"}
```

### `src/main/java/com/messenger/services/`

Пакет бізнес-логіки. Посередник між HTTP API та сховищем. Виконує валідацію, генерує UUID та мітки часу.

| Файл | Метод | Що робить |
|---|---|---|
| `MessageService.java` | `createUser(name)` | Перевіряє ім'я, генерує UUID, зберігає |
| | `createConversation(type)` | Перевіряє тип (direct/group), зберігає |
| | `sendMessage(convId, senderId, text)` | Валідує текст, існування відправника і розмови, зберігає |
| | `getMessages(conversationId)` | Перевіряє існування розмови, повертає повідомлення |
| | `updateStatus(messageId, status)` | Перевіряє допустимість статусу, оновлює |

### `src/main/java/com/messenger/api/`

Пакет HTTP API. Приймає запити, парсить JSON, викликає сервіс, формує відповіді.

| Файл | Відповідальність |
|---|---|
| `Router.java` | Реєстрація маршрутів, диспетчеризація запитів за методом і шляхом, серіалізація відповідей, інтерактивний HTML-інтерфейс на `/` |

**Маршрути:**

| Метод | Маршрут | Обробник |
|---|---|---|
| GET | `/` | Інтерактивний веб-інтерфейс |
| POST | `/users` | `handleUsers` |
| GET | `/users` | `handleUsers` |
| POST | `/conversations` | `handleConversations` |
| GET | `/conversations` | `handleConversations` |
| POST | `/messages` | `handleMessages` |
| GET | `/conversations/{id}/messages` | `handleConversations` |
| PATCH | `/messages/{id}/status` | `handleMessages` |

### `src/main/java/com/messenger/Main.java`

Точка входу. Ініціалізує компоненти у порядку залежностей і запускає сервер:

```java
Database db          = new Database(dataDir);   // 1. сховище
MessageService svc   = new MessageService(db);  // 2. логіка
Router router        = new Router(svc, port);   // 3. API
router.start();
```

### `src/test/java/com/messenger/IntegrationTest.java`

16 інтеграційних тестів. Запускає реальний HTTP-сервер на порту 8765, виконує повний сценарій через HTTP і перевіряє результати.

### `data/`

Директорія для зберігання даних. Створюється автоматично при першому запуску.

| Файл | Вміст |
|---|---|
| `users.jsonl` | Записи користувачів |
| `conversations.jsonl` | Записи розмов |
| `messages.jsonl` | Записи повідомлень зі статусами |

### `postman_collection.json`

Колекція Postman з 8 запитами та автоматичними тестами. Змінні (`userAId`, `convId`, `messageId`) зберігаються між запитами автоматично.

### `run.sh` / `run.bat`

Скрипти для компіляції та запуску на Linux/macOS і Windows:
```bash
./run.sh        # запуск сервера
./run.sh test   # запуск інтеграційних тестів
```

## Архітектура взаємодії компонентів

```
┌─────────────────────────────────────────────────────────┐
│                     HTTP Client                          │
│            (браузер / Postman / curl)                    │
└──────────────────────────┬──────────────────────────────┘
                           │ HTTP запит
                           ▼
┌─────────────────────────────────────────────────────────┐
│                   Router (api/)                          │
│  • маршрутизація за методом і шляхом                     │
│  • парсинг JSON без зовнішніх бібліотек                  │
│  • формування JSON-відповідей                            │
│  • обробка помилок → HTTP 400/404/405/500                │
└──────────────────────────┬──────────────────────────────┘
                           │ виклик методів
                           ▼
┌─────────────────────────────────────────────────────────┐
│              MessageService (services/)                  │
│  • валідація вхідних даних                               │
│  • генерація UUID та timestamp                           │
│  • бізнес-правила (допустимі статуси тощо)               │
└──────────────────────────┬──────────────────────────────┘
                           │ читання / запис
                           ▼
┌─────────────────────────────────────────────────────────┐
│                 Database (storage/)                      │
│  • читання та запис файлів JSON Lines                    │
│  • пошук записів за id                                   │
│  • оновлення статусу (перезапис файлу)                   │
└──────────────────────────┬──────────────────────────────┘
                           │ файловий I/O
                           ▼
┌─────────────────────────────────────────────────────────┐
│                  data/ (файлова система)                 │
│    users.jsonl  conversations.jsonl  messages.jsonl      │
└─────────────────────────────────────────────────────────┘
```
